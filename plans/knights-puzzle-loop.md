## North star

A cozy, account-free, single-player web puzzle game where each level is a
partial board with a start and a goal square. You move a knight (no revisits);
**reaching the goal wins**, and your **score is how many squares you covered
(e.g. 13/17)** — a perfect score visits every square. Imperfect runs can retry
the same board for a better score. Puzzles are generated by random
non-revisiting knight walks, so a perfect path always exists by construction
(**no solver**). Title: **Knight's Puzzle**. Aesthetic: cozy hand-drawn / pixel
grass-field vibe (inspired by enclose.horse, NOT a clone).

What must not be diluted: client-side only · valid-by-construction (never add a
solver) · **win = reach the goal; coverage = score (perfect = all squares)** ·
cutesy but functionally clear, with _which squares are playable_ as the
dominant visual.

> Win-model change (by Nil, after 5d): softened from "cover-all AND end" to
> "reach goal = win, coverage = score, retry for a better score."

## SLICE-5a PICKUP — Engine (authored now)

- **Baseline commit:** `3a09058` (scaffold, all gates green).
- **Goal:** a pure, seeded engine module that the UI and tests both import.
  No DOM, no React — must typecheck under both tsconfigs.
- **Proposed shape** (reviewer may refine):
  - `type Cell = { r: number; c: number }`
  - `type Puzzle = { n; available: boolean[][]; start: Cell; end: Cell;
path: Cell[]; seed: number }`
  - `knightMoves(cell, n): Cell[]` — in-board knight destinations.
  - `makeRng(seed): () => number` — deterministic PRNG (e.g. mulberry32).
  - `generatePuzzle(n, k, seed): Puzzle` — random non-revisiting walk of up
    to `k` steps from a random start; visited cells become `available`;
    `start = path[0]`, `end = path[last]`. Retry/extend to hit a usable
    length; clamp `k <= n*n - 1`; guarantee `path.length >= 2`.
  - `legalMoves(puzzle, current, visited): Cell[]` — knight moves that are
    available and not yet visited.
  - `isWin(puzzle, visited: Cell[]): boolean` — visited covers exactly all
    available cells once, each step a knight move, starts at start, ends at
    end.
- **Load-bearing mechanics / traps:**
  - **Determinism:** same `(n, k, seed)` MUST produce the same puzzle —
    tests depend on it. No `Math.random()` in the engine; thread the RNG.
  - **Dead ends:** a walk can get stuck before `k` steps (all knight
    neighbors visited). That's fine — accept the shorter path; just ensure
    length ≥ 2 and, for a target difficulty, retry a few seeds/starts to
    reach a minimum length. Document the retry policy.
  - **Coordinates:** pick ONE convention (row, col) and keep it everywhere;
    knight deltas are the 8 (±1,±2)/(±2,±1) combos.
- **Acceptance criteria:**
  - Unit tests cover: knightMoves (corner=2, center=8, edge counts);
    generator produces a valid knight path with no revisits, available count
    == path length, correct start/end; determinism (same seed → deep-equal
    puzzle); isWin true for the generator's own path, false for partial /
    wrong-end / non-knight-step; legalMoves filters correctly.
  - `bun run ci && bun run smoke` all green (smoke still passes on skeleton).
- **Decide-with-reviewer:** the `Puzzle` shape and the generator's
  retry/length policy; difficulty-number ranges can wait for 5d.
- **Locked (don't relitigate):** no solver; determinism via seeded RNG;
  engine stays pure (no DOM/React).
